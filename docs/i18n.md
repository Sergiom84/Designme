# Internationalization

Designme Studio uses a single-locale policy for the current product phase.

## Decision

- The user interface is written in Spanish.
- Source code, identifiers, comments, design tokens, data keys, and test names are written in English.
- Generated HTML may render Spanish UI copy by default, while internal metadata and token names remain English.

## Rationale

This keeps the product experience consistent for Spanish-speaking users while preserving a predictable technical surface for contributors, provider prompts, tests, and future integrations.

## Guidelines

- Use Spanish for visible labels, navigation, empty states, buttons, errors, and help text shown to users.
- Use English for TypeScript names, CSS custom properties, token names, comments, filenames, provider IDs, fixtures, and snapshots.
- Do not mix languages inside the same visible UI string unless the term is a proper name, product name, or common technical acronym.
- Prefer stable English token names such as `primary`, `surface`, `radius`, and `shadow` instead of translating token identifiers.
- When adding provider prompts, keep the prompt structure and output contract in English unless the prompt is asking the model to generate user-facing Spanish copy.

## Examples

```ts
const providerLabel = 'Deterministic';
const emptyStateTitle = 'Describe la experiencia que quieres diseñar';
```

```css
:root {
  --color-surface: #ffffff;
  --radius-card: 8px;
}
```
